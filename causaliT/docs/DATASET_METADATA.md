# Dataset Metadata Documentation

The `dataset_metadata.json` file provides variable information and causal structure for training and evaluation. This file is the single source of truth for dataset configuration.

## Overview

For **SCM datasets**, this file is automatically generated by `SCMDataset.generate_ds()`.

For **real-world datasets**, you need to create this file manually following the templates below.

## File Location

```
data/
└── {dataset_name}/
    ├── ds.npz                    # Data file (s, x, y arrays)
    ├── dataset_metadata.json     # ← This file
    ├── dec1_cross_att_mask.csv   # Optional: True DAG masks
    └── dec1_self_att_mask.csv
```

---

## Required Fields by Functionality

### Training (required)

These fields are **required** for training to work:

| Field | Type | Description |
|-------|------|-------------|
| `variable_info.n_source` | int | Number of source variables (S) |
| `variable_info.n_input` | int | Number of input/intermediate variables (X) |
| `variable_info.n_target` | int | Number of target variables (Y) |

### Evaluation Functions (optional)

Different evaluation functions require additional fields:

| Field | Required By | Description |
|-------|-------------|-------------|
| `variable_info.source_labels` | `eval_embed`, `eval_interventions` | List of source variable names: `["S1", "S2", ...]` |
| `variable_info.input_labels` | `eval_embed`, `eval_attention_scores` | List of input variable names: `["X1", "X2", ...]` |
| `variable_info.target_labels` | `eval_attention_scores` | List of target variable names: `["Y1", "Y2", ...]` |
| `causal_structure.direct_edges` | `eval_attention_scores` | List of direct causal edges: `[["S1", "X1"], ...]` |
| `causal_structure.expected_effects` | `eval_interventions` | Expected causal effects for invariance tests |
| `variable_index_map` | `eval_interventions` | Map from variable names to indices |

## Templates

### Minimal Template (Training Only)

Use this for real-world data where you only need training:

```json
{
  "name": "my_dataset",
  "description": "My real-world dataset",
  "variable_info": {
    "n_source": 5,
    "n_input": 3,
    "n_target": 2,
    "source_labels": [],
    "input_labels": [],
    "target_labels": []
  }
}
```

### Full Template (Training + All Evaluations)

Use this for complete functionality including DAG recovery evaluation:

```json
{
  "name": "my_dataset",
  "description": "My dataset with complete causal structure",
  "tags": ["custom", "real-world"],
  
  "variable_info": {
    "n_source": 3,
    "n_input": 2,
    "n_target": 2,
    "source_labels": ["S1", "S2", "S3"],
    "input_labels": ["X1", "X2"],
    "target_labels": ["Y1", "Y2"]
  },
  
  "causal_structure": {
    "direct_edges": [
      ["S1", "X1"],
      ["S2", "X2"],
      ["S3", "X2"],
      ["X2", "X1"],
      ["X1", "Y1"],
      ["X2", "Y2"]
    ],
    "transitive_closure": {
      "S1": ["X1", "Y1"],
      "S2": ["X2", "X1", "Y1", "Y2"],
      "S3": ["X2", "X1", "Y1", "Y2"]
    },
    "expected_effects": {
      "S1": {"X1": true, "X2": false, "Y1": true, "Y2": false},
      "S2": {"X1": true, "X2": true, "Y1": true, "Y2": true},
      "S3": {"X1": true, "X2": true, "Y1": true, "Y2": true}
    }
  },
  
  "variable_index_map": {
    "S1": 1, "S2": 2, "S3": 3,
    "X1": 4, "X2": 5,
    "Y1": 6, "Y2": 7
  },
  
  "feature_indices": {
    "value": 0,
    "variable": 1
  }
}
```

## Field Descriptions

### `variable_info`

| Field | Description |
|-------|-------------|
| `n_source` | Number of source/exogenous variables (length of S sequence) |
| `n_input` | Number of intermediate variables (length of X sequence) |
| `n_target` | Number of target variables (length of Y sequence). Set to `0` if no targets. |
| `source_labels` | Human-readable names for source variables |
| `input_labels` | Human-readable names for input variables |
| `target_labels` | Human-readable names for target variables |

### `causal_structure`

| Field | Description |
|-------|-------------|
| `direct_edges` | List of `[parent, child]` pairs representing direct causal relationships |
| `transitive_closure` | For each source, list all variables it can reach (directly or indirectly) |
| `expected_effects` | For each source S and target X, whether `do(S=s)` should affect X |

### `variable_index_map`

Maps variable names to their integer indices in the data tensor. Used for:
- Creating intervention functions
- Mapping evaluation results to variable names

### `feature_indices`

Specifies which column in the data tensor corresponds to which feature type:

| Field | Default | Description |
|-------|---------|-------------|
| `value` | 0 | Column index for observation values (the actual measurements) |
| `variable` | 1 | Column index for variable IDs (identifies which variable: S1, X1, etc.) |

For SCM datasets, this is always `{value: 0, variable: 1}`.
For real-world datasets, additional features may exist (e.g., timestamps, categorical descriptors).

## Usage in Config Files

With `dataset_metadata.json` in place, you no longer need to hardcode sequence lengths:

**Before (manual):**
```yaml
data:
  dataset: "my_dataset"
  S_seq_len: 5  # Manually maintained
  X_seq_len: 3  # Must match data!
  Y_seq_len: 2  # Must match data!
```

**After (automatic):**
```yaml
data:
  dataset: "my_dataset"
  # All these are auto-populated from dataset_metadata.json:
  S_seq_len: null
  X_seq_len: null
  Y_seq_len: null
  num_embeddings: null  # For nn.Embedding lookup table size
```

The trainer automatically calls `populate_seq_lengths_from_dataset()` which reads these values from the metadata file.

### Auto-Populated Fields

| Config Field | Source | Description |
|--------------|--------|-------------|
| `data.S_seq_len` | `n_source` | Number of source variables |
| `data.X_seq_len` | `n_input` | Number of intermediate variables |
| `data.Y_seq_len` | `n_target` | Number of target variables |
| `data.num_embeddings_X` | `n_input + 1` | For X stage nn.Embedding (non-shared) |
| `data.num_embeddings_Y` | `n_target + 1` | For Y stage nn.Embedding (non-shared) |
| `data.num_embeddings_shared` | `n_source + n_input + n_target + 1` | For shared embedding (all stages) |
| `data.feature_indices.value` | `feature_indices.value` | Column index for observation values |
| `data.feature_indices.variable` | `feature_indices.variable` | Column index for variable IDs |

### Embedding Dimension Usage

**Non-shared embeddings** (each stage has its own lookup table with indices 1,2,3,...):
```yaml
# X embedding uses its own enumeration (X1=1, X2=2, ...)
num_embeddings: ${data.num_embeddings_X}

# Y embedding uses its own enumeration (Y1=1, Y2=2, ...)
num_embeddings: ${data.num_embeddings_Y}
```

**Shared embedding** (one lookup table for all variables):
```yaml
# All variables share one table (S1=1, S2=2, ..., X1=n_source+1, ...)
num_embeddings: ${data.num_embeddings_shared}
```

**OrthogonalMaskEmbedding (S)**: Uses `num_variables: ${data.S_seq_len}` directly.
The `+1` offset for padding is handled internally via `var_id_offset`.

## Error Handling

If `dataset_metadata.json` is missing or invalid, you'll see:

```
DatasetMetadataError: Dataset metadata file not found: data/my_dataset/dataset_metadata.json

The dataset_metadata.json file is required for training.
For SCM datasets, this file is automatically generated by SCMDataset.generate_ds().
For real-world datasets, you need to create this file manually.

See docs/DATASET_METADATA.md for the required format and minimal template.
```

## Generating for SCM Datasets

For SCM datasets, the file is auto-generated:

```python
from scm_ds.datasets import ds_scm1

ds_scm1.generate_ds(
    mode="flat",
    n=50000,
    save_dir="data/scm1_linear_gaussian",
    normalize_method="minmax",
    shared_embedding=False,
)
# dataset_metadata.json is automatically created in save_dir
```

## Validation

To validate your metadata file before training:

```python
from causaliT.training.config_utils import validate_dataset_metadata_for_evaluation

# Validate for training only
metadata = validate_dataset_metadata_for_evaluation(
    data_dir="data",
    dataset="my_dataset",
    required_features=["training"]
)

# Validate for full evaluation suite
metadata = validate_dataset_metadata_for_evaluation(
    data_dir="data",
    dataset="my_dataset",
    required_features=["training", "labels", "edges", "expected_effects"]
)
```
