# SCM Dataset Generation Integration

This document describes the integration between the SCM dataset generator and the uncertainty predictor.

## Overview

The Uncertainty Predictor module supports **two data modes**:

1. **SCM-Generated Data**: Use synthetic datasets generated by the built-in SCM module
2. **External CSV Data**: Upload and use your own CSV datasets

## Architecture

```
AZIMUTH/
└── uncertainty_predictor/            # Uncertainty Predictor (all-in-one)
    ├── scm/                          # SCM Dataset Generator
    │   ├── __init__.py               # Module initialization
    │   ├── generator.py              # Core generator class
    │   ├── generate_dataset.py       # CLI script for generation
    │   ├── README.md                 # SCM module documentation
    │   └── data/                     # Generated datasets
    │       └── scm_dataset.csv       # Default output
    │
    ├── configs/
    │   └── example_config.py         # Configuration (supports both modes)
    ├── train.py                      # Training script (auto-detects data source)
    ├── generate_and_train.py         # Integrated pipeline script
    └── ...
```

## Workflow

### Option A: Auto Mode (Recommended - Easiest)

Set `csv_path='auto'` in config and just run training:

```bash
cd uncertainty_predictor
python train.py
```

The system will:
- ✅ Automatically generate SCM dataset if missing
- ✅ Use existing SCM dataset if present
- ✅ No manual steps required

### Option B: Using External CSV Data

1. Prepare your CSV file with required columns (e.g., `x`, `y`, `z`, `res_1`)
2. Edit `uncertainty_predictor/configs/example_config.py`:
   ```python
   'csv_path': 'path/to/your/data.csv'  # Relative to uncertainty_predictor/
   ```
3. Run training:
   ```bash
   cd uncertainty_predictor
   python train.py
   ```

### Option C: Manual SCM Generation + Training

Generate dataset explicitly, then train:

```bash
cd uncertainty_predictor

# Step 1: Generate dataset
cd scm
python generate_dataset.py --samples 2000

# Step 2: Train
cd ..
python train.py
```

### Option D: Integrated Pipeline (Generate + Train)

```bash
cd uncertainty_predictor
python generate_and_train.py
```

This always generates a fresh SCM dataset and trains on it.

## Configuration

Edit `uncertainty_predictor/configs/example_config.py` to choose your data source:

```python
CONFIG = {
    'data': {
        # DATA SOURCE OPTIONS:

        # Option 1: Auto mode (recommended) - auto-generates if missing
        'csv_path': 'auto',

        # Option 2: Use SCM-generated data explicitly
        # 'csv_path': 'scm/data/scm_dataset.csv',

        # Option 3: Use your external CSV file
        # 'csv_path': 'path/to/your/data.csv',

        'input_columns': ['x', 'y', 'z'],    # Adjust to match your CSV
        'output_columns': ['res_1'],         # Adjust to match your CSV
        # ... other settings
    },
}
```

### Using Your Own Data

If using external CSV:
1. Ensure your CSV has headers matching `input_columns` and `output_columns`
2. Update the column names in config if different
3. Set `csv_path` to your file path (relative to `uncertainty_predictor/`)

## Dataset Format

The SCM generator produces datasets with the following structure:

| Column | Type | Description |
|--------|------|-------------|
| `x` | float | Time/scheduling factor (0-10) |
| `y` | float | Quantity/demand factor (0-10) |
| `z` | float | Resource/capacity factor (0-10) |
| `res_1` | float | Efficiency/performance metric (target) |

### Data Characteristics

- **Non-linear relationships**: Complex interactions between inputs
- **Heteroscedastic noise**: Uncertainty varies across the input space
- **Realistic dynamics**: Simulates real supply chain behavior

The mathematical model is:

```
res_1 = 2.0*x + 3.5*y + 1.5*z
        + 0.5*x*y - 0.3*y*z
        + 0.8*sin(2πx/10)
        - 0.4*x² + 0.2*y²
        + 5.0
        + noise(x, y, z)
```

where `noise(x, y, z)` is heteroscedastic Gaussian noise.

## Customization

### Change Dataset Size

```bash
cd scm
python generate_dataset.py --samples 5000
```

### Change Noise Level

```bash
cd scm
python generate_dataset.py --noise-level 0.2
```

### Use Homoscedastic Noise

```bash
cd scm
python generate_dataset.py --no-heteroscedastic
```

### Modify Training Parameters

Edit `uncertainty_predictor/configs/example_config.py`:

```python
CONFIG = {
    'training': {
        'batch_size': 64,        # Increase batch size
        'epochs': 500,           # More epochs
        'learning_rate': 0.001,  # Higher learning rate
        # ...
    }
}
```

## Programmatic Usage

You can also use the SCM generator programmatically:

```python
from scm import SCMDataGenerator

# Create generator
generator = SCMDataGenerator(
    random_seed=42,
    noise_level=0.15,
    heteroscedastic=True
)

# Generate dataset
df = generator.generate_dataset(n_samples=1000)

# Or generate and save
df = generator.generate_and_save(
    output_path="my_dataset.csv",
    n_samples=1000
)
```

## Benefits of This Integration

1. **Reproducibility**: SCM generator ensures consistent, reproducible datasets
2. **Control**: Fine-grained control over data characteristics
3. **No External Dependencies**: No need for external data files
4. **Flexibility**: Easy to generate datasets with different properties
5. **Testing**: Perfect for testing and experimenting with the uncertainty predictor

## Troubleshooting

### Dataset not found

Ensure the dataset is generated before training:
```bash
cd scm && python generate_dataset.py
```

### Import errors

Install required dependencies:
```bash
pip install -r uncertainty_predictor/requirements.txt
```

### Training fails

Check the configuration file and ensure all paths are correct:
```bash
cd uncertainty_predictor
python -c "from configs.example_config import CONFIG; print(CONFIG['data']['csv_path'])"
```
