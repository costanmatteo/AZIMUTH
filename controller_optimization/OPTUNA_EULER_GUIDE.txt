================================================================================
CONFIGURAZIONE OPTUNA_SWEEP.SH
================================================================================

Nel file optuna_sweep.sh puoi modificare:

#SBATCH --array=0-49         # Numero di trial (0-49 = 50 trial)
#SBATCH --time=00:30:00      # Tempo massimo per trial
REDUCED_EPOCHS=500           # Epoche ridotte per trial veloci
                             # (lascia vuoto per training completo)

================================================================================
IPERPARAMETRI OTTIMIZZATI
================================================================================

- hidden_sizes:           [32,16], [64,32], [128,64], [128,64,32], [256,128,64]
- dropout:                0.0 - 0.4
- use_batchnorm:          True / False
- scenario_embedding_dim: 8 - 64 (step 8)
- learning_rate:          1e-5 - 1e-2 (log scale)

================================================================================
ORDINE CORRETTO
================================================================================

1. Login su Euler
2. (Se necessario) Aggiorna i file
3. (Solo prima volta) Installa Optuna
4. (Se necessario) Esegui train_processes.py
5. Crea lo study Optuna
6. Lancia sbatch optuna_sweep.sh
7. Aspetta che finisca (squeue mostra lista vuota)
8. Genera il report
9. Scarica risultati su Windows

================================================================================
DIFFERENZE DA SWEEP NORMALE
================================================================================

SWEEP NORMALE (sweep.sh):
- Grid search sui seed
- Trial indipendenti
- Nessun database condiviso

OPTUNA (optuna_sweep.sh):
- Ricerca intelligente (TPE sampler)
- Trial comunicano tramite database SQLite
- Pruning automatico (trial scarsi interrotti)
- Visualizzazioni automatiche

================================================================================
TROUBLESHOOTING
================================================================================

Errore "Study not found":
  → Assicurati di aver eseguito --create-study prima

Errore "Database locked":
  → Troppi job simultanei. Aspetta qualche secondo e riprova

Trial tutti pruned:
  → Aumenta REDUCED_EPOCHS in optuna_sweep.sh
  → Oppure rimuovi la riga per usare training completo

Job falliti:
  → Controlla logs/optuna_*.err per dettagli
  → Verifica che train_processes.py sia stato eseguito
